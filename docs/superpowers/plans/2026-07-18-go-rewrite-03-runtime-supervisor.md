# Go Rewrite 03: Runtime Supervisor and Operations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all Bun runtime behavior with a race-safe Go process supervisor, Linux/Windows process-tree control, instance lifecycle orchestration, recovery, resource monitoring, cleanup/auto-update schedulers, and graceful shutdown.

**Architecture:** A central supervisor owns every child process and serializes operations per instance using generation-tagged state. Platform-specific files implement process groups on Linux and Job Objects on Windows. Runtime services depend on repositories and version/filesystem services from Plan 02, while HTTP handlers remain thin adapters.

**Tech Stack:** Go 1.24, `os/exec`, Linux process groups, Windows Job Objects via `golang.org/x/sys/windows`, `github.com/shirou/gopsutil/v4`, Go tests with fake helper processes.

---

## Dependencies and outputs

- Requires Plans 01 and 02.
- Replaces the temporary lifecycle adapter with the real supervisor.
- Does not implement reverse proxying; that is Plan 04.
- Native lifecycle tests are mandatory on both Linux and Windows before this plan is complete.
